# Topic Mode — Design

**วันที่:** 2026-07-23
**สถานะ:** approved, ยังไม่เริ่มโค้ด

เปลี่ยนรูปแบบเกมจาก Wavelength แบบทีม เป็นแบบผลัดกันเป็น **คนเลือก** โดยไม่มีทีม
และบังคับให้คำใบ้ต้องอยู่ในหมวดที่กำหนด

---

## 1. เปลี่ยนอะไร ทำไม

### กติกาเดิม

สุ่ม target ซ่อนไว้ → psychic เห็น → คิด clue อิสระ → ทีมเดา → ทีมตรงข้ามเดาซ้าย/ขวา

### กติกาใหม่

สุ่ม target ซ่อนไว้ → **คนเลือก** เห็น → ดูว่า target อยู่โซนไหนของสเกล →
คิดว่า "ของในหมวดนี้ อันไหนอยู่โซนนั้น" → พิมพ์ชื่อของนั้น → คนอื่นเดาตำแหน่ง

กลไกการหา target เหมือนเดิมทุกอย่าง **สิ่งที่เปลี่ยนคือ clue ถูกบังคับให้อยู่ในหมวด**

### ทำไมต้องเปลี่ยน

Play-test บน production เมื่อ 2026-07-23 พบว่า clue อิสระทำให้เกมพังง่าย —
clue `ประกอบโต๊ะ IKEA` บนสเกล `งานง่าย ↔ งานยาก` ตีความได้กว้างเกินไป
(ขึ้นกับว่าเคยประกอบมาก่อนไหม) ผลคือห่าง 17.4 ได้ 0 คะแนน

การบังคับหมวด (`Movie` → ต้องตอบชื่อหนัง) ทำให้ทุกคนมีกรอบอ้างอิงร่วมกัน
เกมกลับไปวัดสิ่งที่ควรวัด: อ่านใจคนอื่น ไม่ใช่เดาว่าคำนั้นแปลว่าอะไร

### ตัดระบบทีมออก

ทีมทำให้ต้องมี 4 คนขึ้นไปถึงจะสนุก และมีโหมด co-op เป็นทางหนีเมื่อคนน้อย
ซึ่งเป็น code path ที่แทบไม่มีใครเทส — bug 2 ใน 3 ตัวที่เจอตอน play-test
อยู่ใน co-op path พอดี

ผลัดกันเป็นคนเลือกทีละคนใช้ได้ตั้งแต่ 2 คนขึ้นไป ไม่มี path พิเศษ

## 2. โครงสร้างหัวข้อ

3 ชั้น:

| ชั้น | ตัวอย่าง | ใครกำหนด |
|---|---|---|
| **Topic ใหญ่** (category) | `Movie` | deck สุ่ม หรือ คนเลือกพิมพ์ |
| **สเกลซ้าย ↔ ขวา** | `Forgotten ↔ Iconic` | deck สุ่ม หรือ คนเลือกพิมพ์ |
| **Topic ย่อย** (subject) | `Titanic` | **คนเลือกพิมพ์เสมอ** |

**Random:** ระบบสุ่ม Topic ใหญ่ + สเกลให้ กด `สุ่มใหม่` ได้ไม่จำกัด
**Custom:** คนเลือกพิมพ์เองทั้ง 3 ช่อง

Topic ย่อยคือ clue — คนเลือกพิมพ์เองทั้งสองแบบ หลังเห็น target แล้ว

---

## 3. โหมดการเล่น

| | Offline (เครื่องเดียว) | Online (คนละเครื่อง) |
|---|---|---|
| เข็ม | **อันเดียว** ทุกคนช่วยกันขยับ | **คนละอัน** ต่างคนต่างลาก |
| เห็นเข็มคนอื่นตอนเดา | เห็น (เข็มเดียวกัน) | **ซ่อน** เปิดพร้อมกันตอน reveal |
| คะแนน | **กลุ่มเดียว** สะสมรวม | **รายคน** → leaderboard |
| เครื่อง | วางกลางโต๊ะ | ของใครของมัน |
| ส่งเครื่อง | ส่งให้คนเลือก **1 ครั้งต่อรอบ** แล้ววางกลางโต๊ะ | ไม่ต้องส่ง |

Offline ยังต้องมีหน้า pass-the-device ก่อนเผย target ให้คนเลือก
แต่พอพิมพ์ Topic ย่อยเสร็จก็วางกลางโต๊ะ ไม่ต้องส่งอีกในรอบนั้น

## 4. Flow ต่อรอบ

```
pass        offline เท่านั้น — "ส่งเครื่องให้ <คนเลือก>"
            online ข้าม phase นี้ (server เดินผ่านให้เอง เหมือนที่ทำอยู่)
  ↓
topic       คนเลือกกด Random (สุ่มใหม่ได้) หรือ Custom (พิมพ์ 3 ช่อง) → ยืนยัน
  ↓
subject     เห็น target + สเกล → พิมพ์ชื่อที่คิดว่าอยู่โซนนั้น → ส่ง
  ↓
guess       online: ต่างคนต่างลาก ซ่อนกัน / offline: เข็มเดียว
  ↓
reveal      เปิด target + เข็มทุกคนพร้อมกัน + คะแนนรายคน
  ↓
scoreboard  → รอบต่อไป หรือ gameover เมื่อครบจำนวนรอบ
```

Target สุ่มตอนสร้างรอบ แต่คนเลือก**ยังไม่เห็นจนกว่าจะยืนยัน card** เพื่อไม่ให้
ออกแบบ custom spectrum ให้เข้าทาง target ที่เห็นแล้ว

## 5. Data model

```ts
interface TopicCard {
  /** id ของ deck สำหรับกันการ์ดซ้ำ — null เมื่อเป็น custom */
  id: string | null;
  category: string;   // "Movie"
  left: string;       // "Forgotten"   → dial 0
  right: string;      // "Iconic"      → dial 100
  custom: boolean;    // มาจาก deck หรือคนเลือกพิมพ์เอง
}

interface Player {
  id: string;
  name: string;
  score: number;      // ใช้เมื่อ !sharedDial
}

interface Round {
  number: number;
  chooserId: string;
  /** null จนกว่าคนเลือกจะยืนยัน */
  card: TopicCard | null;
  /** ระบบสุ่ม เห็นเฉพาะคนเลือก จนถึง phase reveal */
  target: number;
  /** Topic ย่อย — "" จนกว่าคนเลือกจะส่ง */
  subject: string;
  /** online: key = playerId / offline: key = SHARED_DIAL_KEY */
  guesses: Record<string, number>;
  /** ใครล็อกแล้วบ้าง — key เดียวกับ guesses */
  locked: Record<string, boolean>;
  /** null จนถึง reveal แล้วเป็นคะแนนที่แต่ละคนได้รอบนี้ */
  scores: Record<string, number> | null;
}

interface GameConfig {
  /** จำนวนรอบทั้งเกม 5 / 10 / 15 */
  rounds: number;
  /** วินาทีให้เดา หรือ null = ไม่จับเวลา */
  discussionSeconds: number | null;
  /** true = offline เข็มเดียวคะแนนกลุ่ม */
  sharedDial: boolean;
}

interface GameState {
  config: GameConfig;
  players: Player[];
  phase: Phase;
  round: Round | null;
  /** ใช้เมื่อ sharedDial — คะแนนสะสมของกลุ่ม */
  groupScore: number;
  /**
   * id การ์ดที่ **ยืนยันไปแล้ว** กันซ้ำในเกมเดียว
   * กด `สุ่มใหม่` ไม่นับ — การ์ดจะเข้าลิสต์นี้ตอนคนเลือกกด `ใช้อันนี้` เท่านั้น
   * custom ไม่เข้าลิสต์ (id = null)
   */
  usedCardIds: string[];
}

type Phase =
  | "pass" | "topic" | "subject" | "guess"
  | "reveal" | "scoreboard" | "gameover";
```

`SHARED_DIAL_KEY = "__shared__"` — คีย์คงที่สำหรับโหมดเข็มเดียว

**ลบทิ้ง:** `Team`, `teamId`, `guessTeamId`, `betTeamId`, `BetSide`, `psychicIndex`,
`coop`, `coopRounds`, `targetScore`, `leftRightBet`, `winningTeamId`

## 6. คะแนน

Band เดิม ไม่เปลี่ยน:

| ระยะห่าง `d = \|guess − target\|` | คะแนน |
|---|---|
| `d ≤ 2.5` | 4 |
| `2.5 < d ≤ 7.5` | 3 |
| `7.5 < d ≤ 12.5` | 2 |
| `d > 12.5` | 0 |

**Online (รายคน):**
- คนเดาแต่ละคนได้ตาม band ของเข็มตัวเอง
- **คนเลือกได้ `Math.round(ค่าเฉลี่ยของคนเดาทุกคน)`**
- คนที่ไม่เคยขยับเข็ม → นับที่ 50 ตามค่าเริ่มต้น เข้าสูตรปกติ ไม่ข้าม

**Offline (กลุ่ม):**
- เข็มเดียว ได้คะแนนเดียว บวกเข้า `groupScore`
- คนเลือกไม่มีคะแนนแยก เพราะทั้งกลุ่มใช้คะแนนก้อนเดียว

**จบเกม:** ครบ `config.rounds` รอบ
- online: เรียง leaderboard คนคะแนนสูงสุดชนะ เท่ากันได้ ไม่มี tie-break
- offline: โชว์คะแนนกลุ่มรวม ไม่มีแพ้ชนะ

จำนวนรอบไม่บังคับให้หารลงตัวกับจำนวนคน แต่หน้า lobby / setup จะขึ้นข้อความเตือน
ใต้ปุ่มเลือกจำนวนรอบว่าบางคนจะได้เป็นคนเลือกมากกว่าคนอื่น — เตือนเฉย ๆ กดเริ่มได้อยู่

**ผู้เล่นขั้นต่ำ 2 คน** (คนเลือก 1 + คนเดา 1) ไม่มีขั้นต่ำอื่นแล้ว เพราะไม่มีทีม

## 7. การหมุนคนเลือก

```ts
chooserId = players[(roundNumber - 1) % players.length].id
```

เรียงตามลำดับที่เข้าห้อง ไม่มีทีมให้สลับ ตรงไปตรงมา

**คนเลือกหลุดกลางรอบ (online):** เหมือนเดิม — รอ 30 วิ แล้ว `ABORT_ROUND`
จั่วรอบใหม่ให้คนถัดไป ไม่มีใครได้คะแนน

---

## 8. การซ่อนข้อมูล

3 ชั้นเหมือนเดิม แต่เพิ่มของที่ต้องซ่อน:

| ข้อมูล | ซ่อนจากใคร | จนถึง phase |
|---|---|---|
| `round.target` | ทุกคนยกเว้นคนเลือก | `reveal` |
| `round.guesses[อื่น]` | **ทุกคน** (online) | `reveal` |
| `round.locked` | ไม่ซ่อน — ต้องรู้ว่าใครล็อกแล้ว | — |

`src/server/redact.ts` ยังเป็นจุดเดียวที่ตัดสิน เพิ่มงานตัด `guesses`
ของคนอื่นออก: client เห็นเฉพาะ key ของตัวเอง

`scripts/check-redaction.ts` ต้องเพิ่มเคสว่า guess ของคนอื่นไม่หลุดก่อน reveal

**offline ไม่ต้องซ่อน guesses** เพราะเข็มเดียวทุกคนเห็นอยู่แล้ว

## 10. หน้าจอ

### ใหม่

**`TopicPickerView`** — คนเลือกเท่านั้น 2 แท็บ
- **Random**: โชว์ใบที่สุ่มได้ (`Movie` / `Forgotten ↔ Iconic`) + `สุ่มใหม่` + `ใช้อันนี้`
- **Custom**: 3 ช่อง (Topic ใหญ่ / ซ้าย / ขวา) จำกัด 24 ตัวอักษรต่อช่อง ครบทุกช่องถึงกดต่อได้

**`SubjectView`** — แทน `PsychicView` เห็น target band + card แล้วพิมพ์ Topic ย่อย (จำกัด 40 ตัว)

### แก้

| Component | แก้อะไร |
|---|---|
| `Dial` | เพิ่ม prop `needles?: { value, label, color }[]` รับหลายเข็ม prop `value` เดิมยังใช้ได้สำหรับเข็มเดียว |
| `GuessView` | โชว์ category + สเกล + subject เด่น ตัด prop ที่อ้างถึงทีม |
| `RevealView` | online โชว์เข็มทุกคนพร้อม label ชื่อ + คะแนนรายคน / offline เข็มเดียว |
| `ScoreboardScreen` | online = leaderboard เรียงคะแนน / offline = คะแนนกลุ่มก้อนเดียว |
| `WaitingView` | ข้อความตาม phase: "รอ X เลือกหัวข้อ" / "รอ X คิดคำตอบ" / "รอคนอื่นล็อก" |
| `ScoreBoard` | รับ `Player[]` แทน `Team[]` |
| `Lobby` | ตัด `TeamPicker` เหลือรายชื่อ + ตั้งจำนวนรอบ |

### ลบ

`BetView.tsx`, `TeamPicker.tsx`

### เก็บเดิมไม่แตะ

`TargetBand`, `geometry.ts`, `scoring.ts` (band ไม่เปลี่ยน), `PassDeviceScreen`,
`ui/*`, `useCountdown`, socket client, server rooms/codes, deploy config ทั้งหมด

## 11. Socket events

### เปลี่ยน

| เดิม | ใหม่ |
|---|---|
| `round:clue` | `round:card` (ยืนยัน TopicCard) + `round:subject` (ส่ง Topic ย่อย) |
| `round:guess` | เหมือนเดิม แต่ server เก็บแยกตาม playerId |
| `round:guessMoved` | **ลบ** — เข็มซ่อนกันแล้ว ไม่ต้อง broadcast ระหว่างเดา |
| `round:bet`, `round:lockBet` | **ลบ** |
| `room:setTeam` | **ลบ** |

### ใหม่

`round:reroll` — คนเลือกขอสุ่ม card ใหม่ (phase `topic` เท่านั้น)

### การไปต่อ phase `guess` → `reveal`

ไปเองเมื่อ **คนเดาที่ยัง connected ทุกคนล็อกครบ**
host กดข้ามได้ / timer หมดเวลา server บังคับล็อกที่เหลือ

คนเลือกไม่นับในเงื่อนไขนี้ และไม่มีเข็มของตัวเอง

## 12. ที่ต้องระวัง

- **localStorage bump `v1` → `v2`** ทั้ง `wavelength:local` และ `wavelength:online`
  เกมที่ค้างอยู่ shape เก่าจะ crash ถ้าไม่ bump
- **`pm2 restart` = ห้อง online ที่เล่นอยู่หายหมด** deploy ตอนไม่มีคนเล่น
- ห้ามให้ค่า `target` หรือ `guesses` ของคนอื่นหลุดออก `publicRoom()` ก่อน reveal
- `DESIGN.md` เขียนใหม่ทั้งไฟล์ `PROJECT.md` แก้ตาราง component + socket events

## 13. Testing

- `scripts/check-redaction.ts` — เพิ่มเคส `guesses` ของคนอื่นไม่หลุดก่อน reveal
- ยังไม่มี test runner (ค้างจาก Phase 1) — verify ด้วย typecheck + build + เล่นจริง
- เล่นจริงอย่างน้อย: offline 3 คน 1 เกมจบ, online 3 เครื่อง 1 เกมจบ
- เช็กว่า reroll ไม่ทำให้การ์ดซ้ำในเกมเดียว

## 14. ไม่ทำในรอบนี้ (YAGNI)

- เก็บ custom topic ไว้ใช้ซ้ำ
- แก้ deck ผ่าน UI
- Spectator mode
- Redis-backed rooms
- Tie-break ตอนคะแนนเท่ากัน
