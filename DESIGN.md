# Wavelength (Topic Mode) — Game Design

เกมทายใจแบบ spectrum guessing. ผลัดกันเป็น **คนเลือก** ตั้งหัวข้อแล้วตั้งชื่อของบนสเกล คนอื่นเดาว่าของนั้นอยู่ตรงไหน

> รูปแบบเดิม (แบ่งทีม + psychic + เดาซ้าย/ขวา) ถูกแทนที่ทั้งหมด — spec: [docs/superpowers/specs/2026-07-23-topic-mode-redesign-design.md](docs/superpowers/specs/2026-07-23-topic-mode-redesign-design.md)

---

## 1. โครงหัวข้อ 3 ชั้น

การ์ด 1 ใบ = หมวด + สเกล คนเลือกเติมชื่อของเข้าไปเอง

| ชั้น | ตัวอย่าง | ใครกำหนด |
|---|---|---|
| **หมวด** (category) | `Movie` | deck สุ่ม หรือ คนเลือกพิมพ์ |
| **สเกลซ้าย ↔ ขวา** | `Forgotten ↔ Iconic` | deck สุ่ม หรือ คนเลือกพิมพ์ |
| **ชื่อของ** (subject) | `Titanic` | **คนเลือกพิมพ์เสมอ** |

- **Random**: ระบบสุ่มหมวด + สเกลให้ กด `สุ่มใหม่` ได้ไม่จำกัด (ไม่กินการ์ดจาก deck จนกว่าจะยืนยัน)
- **Custom**: คนเลือกพิมพ์เองทั้ง 3 ช่อง

`left` = ค่า 0 บน dial, `right` = ค่า 100. คู่คำต้องเป็น spectrum ต่อเนื่อง ไม่ใช่ binary

---

## 2. Dial + Target

- แถบครึ่งวงกลม ค่า `0.0 – 100.0`
- Target = เลขทศนิยม 1 ตำแหน่ง สุ่มในช่วง `[12.5, 87.5]` (กัน scoring band ล้นขอบ)
- **คนเลือกเห็น target** แล้วดูว่ามันอยู่โซนไหน → คิดว่า "ของในหมวดนี้ อะไรอยู่โซนนั้น" → พิมพ์ชื่อ
- ชื่อของ **คือตัว clue** — คนอื่นเดาจากชื่อว่าคนเลือกมองว่ามันอยู่ตรงไหน

**สำคัญ:** คนเลือกยืนยันหัวข้อ**ก่อน**เห็น target เสมอ ไม่งั้นเขียน custom spectrum ให้เข้าทาง target ที่เห็นแล้วได้

## 3. ลำดับการเล่น 1 รอบ

```
pass        (offline เท่านั้น) ส่งเครื่องให้คนเลือก
  ↓
topic       คนเลือกกด Random (สุ่มใหม่ได้) หรือ Custom → ยืนยัน
  ↓
subject     เห็น target + สเกล → พิมพ์ชื่อที่คิดว่าอยู่โซนนั้น → ส่ง (target ซ่อนทันที)
  ↓
guess       คนอื่นเดา — online ต่างคนต่างลากเข็มซ่อนกัน / offline เข็มเดียวช่วยกันขยับ
  ↓
reveal      เปิด target + เข็มทุกคนพร้อมกัน + คะแนน
  ↓
scoreboard  → รอบต่อไป หรือ จบเมื่อครบจำนวนรอบ
```

Phase machine: `pass → topic → subject → guess → reveal → scoreboard → (pass | gameover)`

## 4. โหมด: Offline vs Online

| | Offline (เครื่องเดียว) | Online (คนละเครื่อง) |
|---|---|---|
| เข็ม | **อันเดียว** ทุกคนช่วยกันขยับ | **คนละอัน** ต่างคนต่างลาก |
| เห็นเข็มคนอื่นตอนเดา | เห็น (เข็มเดียวกัน) | **ซ่อน** เปิดพร้อมกันตอน reveal |
| คะแนน | **กลุ่มเดียว** สะสมรวม | **รายคน** → leaderboard |
| ส่งเครื่อง | ให้คนเลือก 1 ครั้งต่อรอบ แล้ววางกลางโต๊ะ | ไม่ต้องส่ง |

`sharedDial: true` = offline. Server บังคับ online เป็น `false` เสมอ

## 5. คะแนน

Band เดิม ไม่เปลี่ยน:

```
   2  |  3  |  4  |  3  |  2
-12.5 -7.5  -2.5  +2.5  +7.5 +12.5
              ^ target
```

| ระยะห่าง `d = |guess − target|` | คะแนน |
|---|---|
| `d ≤ 2.5` | **4** |
| `2.5 < d ≤ 7.5` | **3** |
| `7.5 < d ≤ 12.5` | **2** |
| `d > 12.5` | **0** |

**Online (รายคน):** คนเดาแต่ละคนได้ตาม band ของเข็มตัวเอง · **คนเลือกได้ `round(ค่าเฉลี่ยของคนเดาทุกคน)`** — ชื่อที่ไม่มีใครวางถูกเลย คนเลือกได้ 0

**Offline (กลุ่ม):** เข็มเดียว ได้คะแนนเดียว บวกเข้า `groupScore` คนเลือกไม่มีคะแนนแยก

เข็มที่ไม่มีใครแตะ นับที่ 50 ตามค่าเริ่มต้น เข้าสูตรปกติ

**จบเกม:** ครบจำนวนรอบที่ตั้งไว้ (5 / 10 / 15) · online เรียง leaderboard คนคะแนนสูงสุดชนะ · offline โชว์คะแนนกลุ่มรวม

## 6. การหมุนคนเลือก

```
chooserId = players[(roundNumber - 1) % players.length]
```

เรียงตามลำดับที่เข้าห้อง ไม่มีทีมให้สลับ. ผู้เล่นขั้นต่ำ 2 คน (คนเลือก 1 + คนเดา 1)

จำนวนรอบไม่บังคับให้หารลงตัวกับจำนวนคน แต่ UI เตือนถ้าไม่ลงตัว (บางคนได้เลือกมากกว่า)

**คนเลือกหลุดกลางรอบ (online):** รอ 30 วิ แล้ว `ABORT_ROUND` จั่วรอบใหม่ให้คนถัดไป ไม่มีใครได้คะแนน

## 7. การซ่อนข้อมูล — 3 ชั้น

| ข้อมูล | ซ่อนจากใคร | จนถึง phase |
|---|---|---|
| `round.target` | ทุกคนยกเว้นคนเลือก | `reveal` |
| `round.guesses[คนอื่น]` | ทุกคน (online) | `reveal` |
| `round.locked` | ไม่ซ่อน — ต้องรู้ว่าใครล็อกแล้ว | — |

1. **Server ไม่ส่งออก** — `src/server/redact.ts` จุดเดียวที่ตัด `target` + เข็มคนอื่นออกจาก payload. คนเลือกรับ target ผ่าน event `round:target` ยิงเข้า socket ตัวเดียว
2. **Client ไม่ mount** — `<Dial>` ที่ไม่มี prop `target` จะไม่ render scoring band เลย
3. **คนเลือกไม่มีเข็ม** — รู้คำตอบอยู่แล้ว `canGuess = !isChooser`

`npm run check:redaction` เฝ้าชั้น 1 — fail ถ้า target หรือเข็มคนอื่นหลุดก่อน reveal

offline ไม่ต้องซ่อน guesses เพราะเข็มเดียวทุกคนเห็นอยู่แล้ว

## 9. Edge Cases

- **ผู้เล่น < 2 คน** → เริ่มเกมไม่ได้
- **การ์ดหมด deck** → reshuffle เริ่มซ้ำ
- **คนเลือกไม่ส่งชื่อจนหมดเวลา** → (online) timer หมด → reveal ด้วยเข็มที่ค้างอยู่
- **คนเดาไม่ครบล็อก** → รอจนครบ หรือ timer หมด หรือ host กดข้าม
- **คนเลือกหลุด** → ยกเลิกรอบ จั่วใหม่
